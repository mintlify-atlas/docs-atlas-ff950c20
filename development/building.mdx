## Quick Build

```bash
# Clone repository
git clone https://github.com/Gentleman-Programming/gentle-ai.git
cd gentle-ai

# Install dependencies
go mod download

# Build for your platform
go build -o gentle-ai ./cmd/gentle-ai

# Run
./gentle-ai --version
```

## Build Flags

### Development Build

```bash
go build -o gentle-ai ./cmd/gentle-ai
```

- Includes debug symbols
- Fast compile time
- Larger binary size

### Release Build

```bash
go build -ldflags="-s -w -X main.version=v1.0.0" \
  -o gentle-ai ./cmd/gentle-ai
```

**Flags**:
- `-s` - Strip debug symbols
- `-w` - Strip DWARF debug info
- `-X main.version=v1.0.0` - Set version at compile time

**Result**: Smaller binary, faster startup, no debug info.

## Cross-Compilation

### Build for Specific Platform

```bash
# macOS Apple Silicon
GOOS=darwin GOARCH=arm64 go build -o gentle-ai-darwin-arm64 ./cmd/gentle-ai

# macOS Intel
GOOS=darwin GOARCH=amd64 go build -o gentle-ai-darwin-amd64 ./cmd/gentle-ai

# Linux AMD64
GOOS=linux GOARCH=amd64 go build -o gentle-ai-linux-amd64 ./cmd/gentle-ai

# Linux ARM64
GOOS=linux GOARCH=arm64 go build -o gentle-ai-linux-arm64 ./cmd/gentle-ai

# Windows AMD64
GOOS=windows GOARCH=amd64 go build -o gentle-ai-windows-amd64.exe ./cmd/gentle-ai

# Windows ARM64
GOOS=windows GOARCH=arm64 go build -o gentle-ai-windows-arm64.exe ./cmd/gentle-ai
```

### All Platforms at Once

```bash
#!/bin/bash
# build-all.sh

VERSION=${1:-dev}
OUTDIR=./dist

mkdir -p $OUTDIR

platforms=(
  "darwin/amd64"
  "darwin/arm64"
  "linux/amd64"
  "linux/arm64"
  "windows/amd64"
  "windows/arm64"
)

for platform in "${platforms[@]}"; do
  IFS='/' read -r os arch <<< "$platform"
  output="$OUTDIR/gentle-ai-${os}-${arch}"
  
  if [ "$os" = "windows" ]; then
    output="${output}.exe"
  fi
  
  echo "Building for $os/$arch..."
  GOOS=$os GOARCH=$arch go build \
    -ldflags="-s -w -X main.version=$VERSION" \
    -o "$output" \
    ./cmd/gentle-ai
done

echo "Binaries in $OUTDIR/"
ls -lh $OUTDIR/
```

Usage:

```bash
chmod +x build-all.sh
./build-all.sh v1.0.0
```

## GoReleaser (Recommended)

GoReleaser builds, packages, and releases for all platforms automatically.

### Install GoReleaser

```bash
# macOS
brew install goreleaser

# Linux
go install github.com/goreleaser/goreleaser@latest
```

### GoReleaser Config

Gentle AI includes `.goreleaser.yaml`:

```yaml
project_name: gentle-ai

builds:
  - main: ./cmd/gentle-ai
    binary: gentle-ai
    ldflags:
      - -s -w
      - -X main.version={{.Version}}
    goos:
      - darwin
      - linux
      - windows
    goarch:
      - amd64
      - arm64

archives:
  - format: tar.gz
    format_overrides:
      - goos: windows
        format: zip
    files:
      - LICENSE
      - README.md

checksum:
  name_template: 'checksums.txt'

release:
  github:
    owner: Gentleman-Programming
    name: gentle-ai

brews:
  - repository:
      owner: Gentleman-Programming
      name: homebrew-tap
    folder: Formula
    homepage: https://github.com/Gentleman-Programming/gentle-ai
    description: AI Gentle Stack - Ecosystem configurator for AI coding agents
    license: MIT
```

### Build with GoReleaser

```bash
# Build snapshot (no release)
goreleaser build --snapshot --clean

# Build for current platform only
goreleaser build --single-target --snapshot --clean

# Test release process (dry run)
goreleaser release --snapshot --skip=publish --clean
```

Built binaries in `dist/`.

## Creating Releases

### Manual Release

1. **Tag a version**:

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

2. **Build artifacts**:

```bash
goreleaser release --clean
```

3. **Verify release** on GitHub:
   - Binaries uploaded
   - Checksums generated
   - Homebrew formula updated

### GitHub Actions (Automated)

Gentle AI uses GitHub Actions for releases:

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - "v*"

permissions:
  contents: write

jobs:
  goreleaser:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: "1.22"

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          version: "~> v2"
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
```

**Process**:
1. Push a tag: `git push origin v1.0.0`
2. GitHub Actions triggers
3. GoReleaser builds for all platforms
4. Binaries uploaded to GitHub Releases
5. Homebrew formula auto-updated

## Version Management

### Set Version at Build Time

```bash
VERSION=$(git describe --tags --always --dirty)
go build -ldflags="-X main.version=$VERSION" ./cmd/gentle-ai
```

### Version in Code

```go

    "fmt"
    "github.com/gentleman-programming/gentle-ai/internal/app"
)

// Set by GoReleaser via ldflags

func main() {
    app.Version = version
    // ...
}
```

## Troubleshooting

### Build Errors

**Error: `go: module not found`**

```bash
go mod download
go mod tidy
```

**Error: `undefined: embed.FS`**

Update Go version (requires 1.16+):

```bash
go version  # Check current version
# Update Go to 1.22+
```

**Error: Cross-compile fails**

Some packages require CGO for cross-compilation. Gentle AI is CGO-free for portability.

### GoReleaser Issues

**Error: `git tag not found`**

```bash
git fetch --tags
goreleaser release --snapshot  # Test without tags
```

**Error: `HOMEBREW_TAP_TOKEN not set`**

Set GitHub token with repo write access:

```bash
export HOMEBREW_TAP_TOKEN=ghp_xxx
```

### Binary Size

If binary is unexpectedly large:

```bash
# Check symbols
go tool nm gentle-ai | wc -l

# Strip if not already
go build -ldflags="-s -w" ./cmd/gentle-ai

# Check size
ls -lh gentle-ai
```

## Advanced Builds

### Static Binary (Linux)

```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
  go build -a -installsuffix cgo \
  -ldflags="-s -w" \
  -o gentle-ai-static ./cmd/gentle-ai
```

Useful for Docker containers (Alpine, scratch).

### Debug Build

```bash
go build -gcflags="all=-N -l" -o gentle-ai-debug ./cmd/gentle-ai
```

Disables optimizations, enables debugging with `dlv`.

### Profile-Guided Optimization (PGO)

```bash
# 1. Generate profile
go build -o gentle-ai ./cmd/gentle-ai
./gentle-ai install --dry-run  # Collect profile

# 2. Build with profile
go build -pgo=default.pgo -o gentle-ai ./cmd/gentle-ai
```

Can improve performance by 5-10%.
