## Architecture Overview

Gentle AI is built on three core principles:

1. **Agent Agnostic** - Adapter pattern abstracts agent differences
2. **Dependency-Aware** - Graph-based component ordering
3. **Safe by Default** - Backup before changes, rollback on failure

## Design Patterns

### 1. Adapter Pattern (Agents)

Every agent implements the `Adapter` interface:

```go
type Adapter interface {
    // Identity
    Agent() model.AgentID
    Tier() model.SupportTier

    // Detection
    Detect(ctx context.Context, homeDir string) (
        installed bool, binaryPath string, 
        configPath string, configFound bool, err error)

    // Config paths
    GlobalConfigDir(homeDir string) string
    SkillsDir(homeDir string) string
    SystemPromptFile(homeDir string) string
    SettingsPath(homeDir string) string

    // Config strategies
    SystemPromptStrategy() model.SystemPromptStrategy
    MCPStrategy() model.MCPStrategy

    // Feature support
    SupportsSkills() bool
    SupportsSystemPrompt() bool
    SupportsMCP() bool
    SupportsOutputStyles() bool
    SupportsSlashCommands() bool
}
```

**Example: Claude Code Adapter**

```go
type ClaudeAdapter struct{}

func (a ClaudeAdapter) SkillsDir(homeDir string) string {
    return filepath.Join(homeDir, ".claude", "skills")
}

func (a ClaudeAdapter) SystemPromptStrategy() model.SystemPromptStrategy {
    return model.StrategyMarkdownSections // Use markers
}
```

**Why This Matters**

Components use adapters without knowing which agent:

```go
// Component code is agent-agnostic
func (i *SkillsInstaller) Install(
    ctx context.Context, 
    adapter agents.Adapter,
) error {
    skillsDir := adapter.SkillsDir(homeDir)
    // Write skills to adapter-provided path
}
```

Adding a new agent? Just implement `Adapter` - no component changes needed.

### 2. Dependency Graph (Components)

Components declare dependencies:

```go
// internal/catalog/dependencies.go
var Dependencies = map[model.ComponentID][]model.ComponentID{
    model.ComponentSDD: {
        model.ComponentEngram,  // SDD needs Engram
        model.ComponentSkills,  // SDD needs Skills
    },
    model.ComponentPersona: {
        model.ComponentEngram,  // Persona references Engram
    },
}
```

The planner topologically sorts components:

```go
// internal/planner/order.go
func TopologicalSort(graph DependencyGraph) ([]model.ComponentID, error) {
    // Returns: [engram, skills, context7, sdd, persona, ...]
}
```

Result: components install in the correct order automatically.

### 3. Staged Pipeline (Execution)

Execution flows through stages:

```go
type StagePlan struct {
    Prepare []Step  // Validation, detection
    Apply   []Step  // Install, write files
}

type Step interface {
    ID() string
    Execute(ctx context.Context) error
    Rollback(ctx context.Context) error  // Undo this step
}
```

**Stage 1: PREPARE**
- Detect agents (binary + config paths)
- Validate dependencies (git, npm, node)
- Check disk space, permissions
- **No side effects** - safe to run repeatedly

**Stage 2: APPLY**
- Create backups of existing configs
- Install components in dependency order
- Write skills, inject system prompts, configure MCP
- **Side effects** - modifies filesystem

**Stage 3: ROLLBACK** (on failure)
- Restore files from backup manifest
- Clean up partial writes
- Report what was rolled back

### 4. Marker-Based File Merging

Inject content without clobbering user edits:

```markdown
<!-- User's existing content -->
My custom instructions...

<!-- gentle-ai:persona-start -->
You are the Gentleman AI persona...
<!-- gentle-ai:persona-end -->

<!-- More user content -->
```

**Implementation** (internal/components/filemerge/):

```go
func InjectSection(filePath, markerID, content string) error {
    existing := readFile(filePath)
    
    // Replace content between markers
    updated := replaceMarkerSection(existing, markerID, content)
    
    writeFile(filePath, updated)
}
```

**Benefits**:
- Preserves user customizations
- Idempotent updates (re-run safely)
- Clean uninstall (remove markers + content)

## Component Architecture

### Component Interface

Each component implements:

```go
type Component interface {
    ID() model.ComponentID
    Install(ctx context.Context, adapter agents.Adapter) error
    Verify(ctx context.Context, adapter agents.Adapter) error
}
```

### Example: Skills Component

```go
// internal/components/skills/installer.go
type Installer struct {
    assets  assets.SkillProvider
    logger  Logger
}

func (i *Installer) Install(
    ctx context.Context,
    adapter agents.Adapter,
) error {
    // 1. Get skills directory from adapter
    skillsDir := adapter.SkillsDir(homeDir)
    
    // 2. Create directory
    os.MkdirAll(skillsDir, 0755)
    
    // 3. Write skill files
    for _, skillID := range selection.Skills {
        content := i.assets.GetSkill(skillID)
        filePath := filepath.Join(skillsDir, skillID + ".md")
        os.WriteFile(filePath, content, 0644)
    }
    
    // 4. Handle agent-specific formats
    if adapter.SupportsSlashCommands() {
        // OpenCode: also write slash commands
        i.writeSlashCommands(adapter)
    }
    
    return nil
}
```

## Key Interfaces

### Adapter (Agent Integration)

See `internal/agents/interface.go`

- **Purpose**: Abstract agent differences
- **Used by**: All components
- **Implementations**: claude/, opencode/, gemini/, cursor/, vscode/

### Component (Installation)

See component packages under `internal/components/`

- **Purpose**: Install/configure a single component
- **Used by**: Pipeline orchestrator
- **Implementations**: engram/, sdd/, skills/, mcp/, persona/, etc.

### Step (Pipeline)

See `internal/pipeline/stages.go`

- **Purpose**: Atomic unit of work with rollback support
- **Used by**: Orchestrator
- **Examples**: DetectAgentStep, InstallComponentStep, BackupStep

## Performance Considerations

### Parallel Detection

Agents are detected in parallel:

```go
for _, agentID := range selection.Agents {
    go func(id model.AgentID) {
        adapter := agents.NewAdapter(id)
        info := adapter.Detect(ctx, homeDir)
        results <- info
    }(agentID)
}
```

### Lazy Asset Loading

Skills are embedded but loaded on-demand:

```go
//go:embed skills/*.md
var skillsFS embed.FS

func GetSkill(id SkillID) []byte {
    return skillsFS.ReadFile(fmt.Sprintf("skills/%s.md", id))
}
```
