# Permissions

The Permissions component configures security guardrails for AI agents, setting safe defaults that auto-approve most operations while blocking dangerous commands and sensitive file access.

## What is Permissions?

Permissions controls:
- **Tool call approval**: Which operations require user confirmation
- **Command blocking**: Deny dangerous bash commands
- **File access**: Prevent reading secrets and credentials
- **Git safety**: Ask before destructive git operations

Think of it as "safe mode" for AI agents—letting them work autonomously while protecting you from accidents.

## Philosophy: Security-First Defaults

The permissions system follows these principles:

1. **Auto-approve safe operations**: File edits, reads, searches
2. **Ask for destructive actions**: Git force push, hard reset, commit
3. **Block dangerous commands**: `rm -rf /`, credential files
4. **Explicit > Implicit**: Better to ask than cause damage

## Permission Overlays

### Claude Code

```json
{
  "permissions": {
    "defaultMode": "bypassPermissions",
    "deny": [
      "rm -rf /",
      "sudo rm -rf /",
      ".env"
    ]
  }
}
```

**Behavior**: Auto-accept all tool calls EXCEPT the explicit deny list.

Valid modes: `acceptEdits`, `bypassPermissions`, `default`, `dontAsk`, `plan`

## Pattern Matching (OpenCode)

OpenCode's permission system supports glob patterns:

| Pattern | Matches | Example |
|---------|---------|----------|
| `*` | Everything | `bash: { "*": "allow" }` |
| `git push *` | Git push with any args | `git push origin main` |
| `*.env` | Files ending in .env | `production.env` |
| `**/.env` | .env in any subdirectory | `src/config/.env` |
| `**/secrets/**` | Any file in secrets directory | `config/secrets/api-key.txt` |

### Priority Order

More specific patterns override wildcards:

```json
{
  "bash": {
    "*": "allow",              // Default: allow everything
    "git push --force *": "ask" // Override: ask for force push
  }
}
```

## Installation Behavior

Permissions component:

1. **Check agent support**: Only runs if agent has a settings file
2. **Get agent overlay**: Retrieve agent-specific permission config
3. **Merge into settings**: Deep merge the overlay into existing settings.json
4. **Report changes**: Return whether file was modified

Source: `internal/components/permissions/inject.go:90-107`

```go
func Inject(homeDir string, adapter agents.Adapter) (InjectionResult, error) {
    settingsPath := adapter.SettingsPath(homeDir)
    if settingsPath == "" {
        return InjectionResult{}, nil // Agent doesn't use settings file
    }

    overlay := agentOverlay(adapter.Agent())
    if overlay == nil {
        return InjectionResult{}, nil // Agent doesn't support permission injection
    }

    writeResult, err := mergeJSONFile(settingsPath, overlay)
    return InjectionResult{
        Changed: writeResult.Changed,
        Files:   []string{settingsPath},
    }, err
}
```

## Why Auto-Approve?

You might wonder: "Isn't auto-approval dangerous?"

### The Token Cost Problem

Every permission prompt:
- Interrupts the agent's flow
- Requires a full context reload after user input
- Consumes 2-3x tokens per operation
- Breaks agent reasoning mid-task

For a typical SDD workflow (50+ file operations), asking for each edit would:
- Cost 100+ interruptions
- Consume 10-50K extra tokens
- Make the agent nearly unusable

### The Security Solution

Instead of blocking everything, Gentle AI:
1. **Allows safe operations**: File edits, reads (except secrets)
2. **Blocks dangerous patterns**: Destructive commands, credential access
3. **Asks for risky actions**: Git operations that rewrite history

This gives you **90% autonomy** with **99% safety**.

## Customizing Permissions

### Adding More Deny Patterns (OpenCode)

Edit `~/.config/opencode/opencode.json`:

```json
{
  "permission": {
    "bash": {
      "npm publish *": "ask",     // Ask before publishing packages
      "docker rm *": "ask"         // Ask before removing containers
    },
    "read": {
      "**/private/**": "deny",     // Block private directory
      "*.key": "deny"              // Block key files
    }
  }
}
```

### Disabling Auto-Approval

If you prefer manual control:

**Claude Code**: Change `"defaultMode"` to `"default"`

**OpenCode**: Change `"*": "allow"` to `"*": "ask"` for bash and read

**Gemini CLI**: Change `"defaultApprovalMode"` to `"always-ask"`

**VS Code Copilot**: Set `"chat.tools.autoApprove": false`

## Merge Logic

Permissions uses deep JSON merging:

```go
func mergeJSONFile(path string, overlay []byte) (WriteResult, error) {
    // Read existing settings (or treat as empty if not found)
    baseJSON, err := osReadFile(path)
    if err != nil {
        return WriteResult{}, err
    }

    // Deep merge overlay into base
    merged, err := filemerge.MergeJSONObjects(baseJSON, overlay)
    if err != nil {
        return WriteResult{}, err
    }

    // Write atomically to prevent corruption
    return filemerge.WriteFileAtomic(path, merged, 0o644)
}
```

**Behavior**:
- Existing settings are preserved
- New keys from overlay are added
- Overlapping keys use overlay values
- Atomic write prevents partial updates

## FAQ

<Accordion title="Can the agent still commit secrets if I don't block writes?">
  Yes, if you allow git commits without asking. Best practice: Keep `git commit` in the "ask" list and use `.gitignore` to prevent staging secrets.
</Accordion>

<Accordion title="What if I want to read a .env file?">
  Temporarily edit your permissions config to allow it, or use `cat .env` in your terminal instead of asking the agent.
</Accordion>

<Accordion title="Why doesn't Gentle AI configure Cursor permissions?">
  Cursor manages permissions via `cli-config.json`, which has a different schema. Gentle AI focuses on agents with standardized settings files.
</Accordion>

<Accordion title="Can I disable permissions entirely?">
  Yes. Edit your agent's settings file and remove the permission keys Gentle AI added. Your agent will revert to its default permission behavior.
</Accordion>

<Accordion title="Do permission denials show up in logs?">
  Depends on the agent. OpenCode shows permission denials in the UI. Claude Code may silently skip denied operations. Check your agent's documentation.
</Accordion>
