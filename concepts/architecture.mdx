## Project Structure

Gentle AI is a Go CLI tool with a clean internal architecture:

```
cmd/gentle-ai/             CLI entrypoint
internal/
  app/                     Command dispatch + runtime wiring
  model/                   Domain types (agents, components, skills, presets)
  catalog/                 Registry definitions
  system/                  OS/distro detection, dependency checks
  cli/                     Install flags, validation, orchestration
  planner/                 Dependency graph, resolution, ordering
  installcmd/              Platform-aware command resolver
  pipeline/                Staged execution + rollback orchestration
  backup/                  Config snapshot + restore
  assets/                  Embedded skill files + persona templates
  components/              Per-component install/inject logic
  agents/                  Agent adapters (config strategy per agent)
  verify/                  Post-apply health checks + reporting
  tui/                     Bubbletea TUI (Rose Pine theme)
scripts/                   Installer scripts (bash + PowerShell)
e2e/                       Docker-based E2E tests
testdata/                  Golden test fixtures
```

## Core Layers

### Command Layer (`cmd/` + `internal/app/`)

The entry point that dispatches to commands:

```go
// cmd/gentle-ai/main.go
package main

import (
    "github.com/gentleman-programming/gentle-ai/internal/app"
)

var version = "dev" // Set by GoReleaser

func main() {
    app.Version = version
    if err := app.Run(); err != nil {
        fmt.Fprintf(os.Stderr, "Error: %v\n", err)
        os.Exit(1)
    }
}
```

The `app` package routes to:
- **No args** → TUI mode
- `install` → CLI install mode
- `update` → Update checker
- `version` → Version printer

### System Layer (`internal/system/`)

OS and dependency detection:

<CodeGroup>

```go Detection
// internal/system/detect.go
func Detect(ctx context.Context) (DetectionResult, error) {
    homeDir, _ := os.UserHomeDir()
    tools := DetectTools(ctx, []string{"git", "curl", "brew", "node"})
    configs := ScanConfigs(homeDir)
    
    profile := resolvePlatformProfile(runtime.GOOS, osRelease, tools)
    dependencies := DetectDependencies(ctx, profile)
    
    return DetectionResult{
        System:       SystemInfo{OS: runtime.GOOS, Profile: profile},
        Tools:        tools,
        Configs:      configs,
        Dependencies: dependencies,
    }, nil
}
```

```go Platform Profiles
// Platform-specific package managers
type PlatformProfile struct {
    OS             string
    LinuxDistro    string  // ubuntu, debian, arch
    PackageManager string  // brew, apt, pacman, winget
    NpmWritable    bool    // true if npm prefix is user-writable
}
```

</CodeGroup>

### Planner Layer (`internal/planner/`)

Dependency resolution and ordering:

<Steps>
  <Step title="Build Dependency Graph">
    Each component declares its dependencies:
    
    ```go
    graph := map[ComponentID][]ComponentID{
        ComponentSDD:     {ComponentEngram},
        ComponentSkills:  {ComponentSDD},
        ComponentContext7: {},
        ComponentPersona: {},
        ComponentGGA:     {},
    }
    ```
  </Step>
  
  <Step title="Resolve Selection">
    Given user selection, resolve transitive dependencies:
    
    - User selects: `[skills, context7]`
    - Planner resolves: `[engram, sdd, skills, context7]`
    - Ordered: `[engram, sdd, skills, context7]`
  </Step>
  
  <Step title="Generate Review">
    Build human-readable review payload:
    
    ```go
    type ReviewPayload struct {
        Agents           []AgentSummary
        Components       []ComponentWithDeps
        MissingDeps      []Dependency
        InstallCommands  []string
    }
    ```
  </Step>
</Steps>

### Pipeline Layer (`internal/pipeline/`)

Staged execution with rollback:

```go
type StagePlan struct {
    Prepare []Step  // Dependency checks, backup
    Apply   []Step  // Agent config, component injection
}

type Orchestrator struct {
    rollbackPolicy RollbackPolicy
    failurePolicy  FailurePolicy
    progressFunc   ProgressFunc
}

func (o *Orchestrator) Execute(plan StagePlan) ExecutionResult
```

**Execution flow**:
1. **Prepare stage**: Check dependencies, create backup
2. **Apply stage**: Configure agents, inject components
3. **Rollback** (on failure): Restore from backup

## Data Flow

<Steps>
  <Step title="User Input">
    TUI or CLI flags capture:
    - Selected agents
    - Chosen preset/components
    - Persona mode
  </Step>
  
  <Step title="Detection">
    System layer scans:
    - OS and architecture (source: `/internal/system/detect.go:34`)
    - Package managers
    - Existing agent configs
    - Dependencies
  </Step>
  
  <Step title="Planning">
    Planner resolves:
    - Transitive dependencies (source: `/internal/planner/resolver.go`)
    - Install order
    - Review payload
  </Step>
  
  <Step title="Execution">
    Pipeline orchestrates:
    - Backup creation (source: `/internal/backup/`)
    - Dependency installation
    - Agent configuration
    - Component injection
  </Step>
  
  <Step title="Verification">
    Verify layer checks:
    - File existence
    - JSON validity
    - Executable permissions
  </Step>
</Steps>
